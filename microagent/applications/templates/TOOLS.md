# 工具使用指南

> 本文件描述 Agent 可用的工具及其使用方法。
> 了解这些工具可以帮助你更好地与 Agent 协作。

---

## 文件系统工具

### filesystem_read
**功能**：读取文件内容

**适用场景**：
- 查看文件内容
- 分析代码逻辑
- 提取文档信息

**使用示例**：
```
读取 README.md 文件
查看 src/index.ts 的内容
```

**注意事项**：
- 仅能访问工作目录内的文件
- 大文件可能需要分段读取

---

### filesystem_write
**功能**：创建或覆盖写入文件

**适用场景**：
- 创建新文件
- 保存生成的内容
- 导出数据

**使用示例**：
```
创建一个名为 notes.md 的文件，内容是...
把这段代码保存到 output.ts
```

**注意事项**：
- 会覆盖已存在的文件
- 不会自动创建父目录

---

### filesystem_edit
**功能**：编辑现有文件（精确替换）

**适用场景**：
- 修改代码中的特定部分
- 更新配置项
- 修复文档错误

**使用示例**：
```
把 index.ts 中的 function foo 改成 function bar
在 config.yaml 中添加一个新的配置项
```

**注意事项**：
- 需要提供精确的原文和替换内容
- 支持正则表达式匹配

---

### filesystem_list
**功能**：列出目录内容

**适用场景**：
- 浏览项目结构
- 查找特定文件
- 了解目录组织

**使用示例**：
```
列出当前目录的文件
查看 src 目录下有什么
```

---

## 命令执行工具

### shell_execute
**功能**：执行系统命令

**适用场景**：
- 运行构建命令
- 执行测试
- 安装依赖

**使用示例**：
```
运行 npm install
执行 bun test
启动开发服务器
```

**注意事项**：
- 仅在安全命令白名单内执行
- 危险命令会被拒绝
- 执行前可能需要确认

---

## 网络工具

### web_search
**功能**：搜索网络信息

**适用场景**：
- 查询最新资料
- 了解技术动态
- 解决具体问题

**使用示例**：
```
搜索 TypeScript 5.0 的新特性
查找 Next.js 部署最佳实践
```

**注意事项**：
- 结果可能包含过时信息
- 需要验证来源可靠性

---

### web_fetch
**功能**：获取网页内容

**适用场景**：
- 阅读在线文档
- 获取 API 文档
- 抓取页面内容

**使用示例**：
```
获取 https://example.com 的内容
读取 MDN 上关于 Promise 的文档
```

---

## 使用最佳实践

### 1. 明确指定目标
好的请求：
```
读取 package.json 文件，找出所有的依赖项
```

模糊的请求：
```
看看那个文件
```

### 2. 分步骤处理复杂任务
```
1. 先读取配置文件
2. 分析当前的配置结构
3. 添加新的配置项
4. 验证配置是否正确
```

### 3. 关键操作前确认
```
我会删除 node_modules 目录，这需要大约 1 分钟，确认执行吗？
```

### 4. 利用上下文
```
基于刚才读取的代码，继续完善错误处理逻辑
```

## 工具权限说明

| 工具 | 权限级别 | 需要确认 |
|------|----------|----------|
| 读取文件 | 低 | 否 |
| 写入文件 | 中 | 新文件否，覆盖是 |
| 删除文件 | 高 | 是 |
| 执行命令 | 中-高 | 取决于命令 |
