# 心跳任务

> 本文件定义定时执行的任务和检查项。
> Agent 会在每次心跳时检查这些任务。

---

## 心跳机制说明

**心跳间隔**：默认 30 分钟（可在 settings.yaml 中配置）

**心跳触发**：
- 空闲时定期执行
- 检查是否有待处理任务
- 执行维护和提醒功能

## 维护任务

<!-- 定义需要定期执行的维护操作 -->

### 日志维护
- 自动清理超过 7 天的日志
- 压缩大型日志文件
- 导出重要日志到归档

### 会话维护
- 清理过期会话数据
- 压缩历史会话记录
- 备份重要对话

### 缓存维护
- 清理过期缓存
- 检查缓存大小
- 重建缓存索引

## 自定义任务

> 在下方添加你的自定义心跳任务：

```yaml
# 示例：定时检查 GitHub Issues
- name: check_github_issues
  interval: 1h
  action: |
    检查指定仓库的新 Issues
    如果有新 Issue，通知用户
```
